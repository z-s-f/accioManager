# Accio Manager

Accio Manager 是一个本地运行的 Accio 多账号管理面板，提供账号资料查看、凭证保存、配额追踪、活跃账号识别以及本地 Profile 切换能力，适合个人在本机环境中管理和维护多个 Accio 使用身份。

## 项目简介

这个项目基于 `Node.js + Express` 构建，前端为静态页面，后端负责读取本地运行环境中的账号信息、Accio 应用状态和缓存数据，并通过 Web 面板统一展示。

当前项目重点在于：

- 汇总本地账号列表与活跃账号状态
- 保存账号本地 Profile 与认证文件
- 展示账号配额、认证有效期与基础元数据
- 提供本地切换和 OAuth 切换相关能力
- 将敏感运行数据保存在本地 `data/` 目录，避免直接进入版本库

## 功能概览

- 多账号总览：展示账号数量、任务数量、会话数量等统计信息
- 活跃账号检测：结合本地日志与元数据识别当前账号
- 凭证管理：保存并展示 Token、Cookie、设备信息等认证数据
- Profile 管理：按账号保存本地 Profile 快照，支持重复切换
- 配额跟踪：记录账号配额消耗、刷新倒计时和检查时间
- 本地面板：通过浏览器访问本机服务进行操作

## 技术结构

- 后端入口：`server.js`
- 前端页面：`public/index.html`
- 前端逻辑：`public/app.js`
- 前端样式：`public/styles.css`
- 本地运行数据：`data/`

## 快速开始

### 环境要求

- Node.js 18 及以上
- macOS 本地环境
- 已安装并使用过 Accio 桌面应用

### 下载 Accio

如果本机尚未安装 Accio，可先前往官方页面下载：

- Accio 下载地址：https://www.accio.com/work

### 安装依赖

```bash
npm install
```

### 启动项目

```bash
npm run dev
```

启动后访问：

```text
http://localhost:3456
```

## 使用指南

建议按下面顺序完成首次使用：

1. 先安装并启动 Accio 桌面应用，确认你已经在本机成功登录至少一个账号。
2. 在项目目录执行 `npm install` 安装依赖。
3. 执行 `npm run dev` 启动 Accio Manager。
4. 在浏览器打开 `http://localhost:3456` 进入管理面板。
5. 进入总览页，确认系统已经识别当前活跃账号和本地运行状态。

如果系统没有自动识别完整账号信息，可以继续使用下面两种方式补充认证数据：

- 方式一：使用页面里的 OAuth 授权链接，完成授权后把回调 URL 提交到面板
- 方式二：使用“导入授权链接”功能，手动粘贴 `http://127.0.0.1:4097/auth/callback?...` 回调地址

更完整的图文说明见：[docs/USAGE.md](./docs/USAGE.md)

## 系统操作指南

### 1. 总览页

- 查看当前活跃账号、账号总数、任务数、会话数等基础统计
- 使用“手动同步”按钮刷新最新本地状态
- 使用授权区域生成或打开 OAuth 链接，并提交回调 URL
- 在“账号快速切换”区域点击账号卡片切换当前活动账号

### 2. 账号管理

- 查看所有账号的标签、基础资料和当前状态
- 检查账号是否已经保存本地 Profile
- 查看各账号最近更新时间和相关元数据

### 3. 凭证中心

- 查看 Access Token、Refresh Token、Cookie 等认证信息的保存状态
- 手动补充或更新账号凭证信息
- 检查认证是否临近过期，便于及时刷新

### 4. 配额管理

- 查看每个账号的已使用量、总配额和剩余额度
- 手动更新单个账号的配额数据
- 使用批量刷新功能从远端接口同步可获取的最新配额

### 5. Profile 保存与切换

- 在账号存在有效本地登录状态时保存 Profile
- 为不同账号保留独立本地快照，减少重复登录
- 在支持的情况下通过本地 Profile 或 OAuth 方式执行账号切换

### 6. 常见使用建议

- 首次公开演示前，先检查页面中是否展示了真实 Token、Cookie 或账号邮箱
- 切换账号前，尽量保证 Accio 客户端处于稳定状态
- 如果授权失败，优先检查本机是否已启动 Accio，以及本地回调端口是否可用
- 如果数据显示异常，可先点击“手动同步”再重新检查

## 本地数据说明

应用运行时会在项目根目录创建或使用 `data/` 目录，常见内容包括：

- `data/accounts_meta.json`：账号元信息、认证信息、配额信息
- `data/profiles/`：每个账号独立保存的本地 Profile、Cookie、凭证文件

同时，程序还会读取用户目录中的 Accio 本地数据，例如：

- `~/.accio/accounts`
- `~/.accio/settings.jsonc`
- `~/.accio/logs/sdk.log`
- `~/Library/Application Support/Accio`

这些文件可能包含敏感账号信息、登录态、设备标识或本地缓存，默认仅建议在个人本机环境中使用。
