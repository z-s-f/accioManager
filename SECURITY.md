# Security Policy

## Sensitive Data

This project is designed to work with local account metadata, profile snapshots, cookies, tokens and other credential-related artifacts.

Do not include any of the following in public issues, pull requests, screenshots or logs:

- access tokens
- refresh tokens
- cookies
- local profile files
- account metadata exports
- device identifiers

Before sharing logs or code snippets, redact all sensitive values.

## Reporting a Vulnerability

If you discover a security issue in this project, do not disclose it publicly first.

- Contact the repository owner through a private channel
- Provide reproduction steps, impact and affected files if possible
- Wait for the maintainer to assess the report before opening a public issue

If no private contact channel is listed yet, open a minimal public issue without sensitive details and ask the maintainer for a private reporting method.
